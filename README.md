## InstalaciГіn

Abre Termux y pega este comando:

```bash
curl -fsSL https://raw.githubusercontent.com/Honkonx/termux-ai-stack/main/instalar.sh \
  -o instalar.sh && bash instalar.sh
```

> **Requisito:** [Termux desde F-Droid](https://f-droid.org/packages/com.termux/) вҖ” no uses la versiГіn de Play Store, estГЎ desactualizada y sin mantenimiento.

El script se encarga de todo: permisos, dependencias, tema visual, descarga de scripts y configuraciГіn del dashboard. Tiene checkpoints automГЎticos вҖ” si algo falla, vuГ©lvelo a ejecutar y continГәa desde donde quedГі.

## Backup y Restore

El sistema de backup genera archivos `.tar.xz` por mГіdulo, verifica integridad con SHA256 y permite restaurar desde GitHub Releases o desde tu propio backup.

### Hacer backup

```bash
bash ~/backup.sh                   # backup completo (6 partes + checksums)
bash ~/backup.sh --module ollama   # solo Ollama
bash ~/backup.sh --module claude   # solo Claude Code
bash ~/backup.sh --module n8n      # solo n8n + cloudflared
bash ~/backup.sh --module expo     # solo EAS CLI
bash ~/backup.sh --module base     # solo scripts y configuraciГіn base
```

Los archivos se guardan en `/sdcard/Download/termux-ai-stack-releases/`.

### Restaurar

```bash
bash ~/restore.sh                               # menГә interactivo
bash ~/restore.sh --module ollama               # mГіdulo especГӯfico
bash ~/restore.sh --module all --source github  # todo desde GitHub Releases
bash ~/restore.sh --module n8n --source local   # desde tu backup propio
```

### Partes del backup

| Parte | Contenido | TamaГұo aprox. |
|-------|-----------|---------------|
| part1-termux-base | .bashrc + scripts + .termux | ~120KB |
| part2-claude-code | @anthropic-ai/claude-code completo | ~12KB |
| part3-eas-expo | eas-cli + credenciales ~/.expo | ~12MB |
| part4-ollama | binario + libs (sin modelos) | ~9MB |
| part5-n8n-data | n8n + cloudflared + workflows | ~15MB |
| part6-proot-debian | rootfs Debian completo | ~834MB |

> вҡ пёҸ Los modelos de Ollama NO se incluyen en el backup. DescГЎrgalos de nuevo con `ollama pull`.

## MГіdulo: n8n

AutomatizaciГіn de workflows. Corre dentro de proot Debian para tener glibc real (necesario para node-gyp). Cloudflared provee una URL pГәblica desde internet sin configurar puertos.

```
вң… proot-distro + Debian Bookworm ARM64
вң… Node.js 20 LTS + n8n + cloudflared (dentro del proot)
вң… TГәnel cloudflared вҶ’ URL pГәblica (dominio fijo o URL temporal)
вң… Webhook Telegram funcionando (WEBHOOK_URL + N8N_PROTOCOL=https)
вң… Scripts de control: start / stop / url / status / logs
вң… SesiГіn tmux "n8n-server" en background
```

**Variables requeridas para webhooks Telegram:**
```bash
# En ~/.env_n8n (configurable desde menГә вҶ’ n8n вҶ’ [8]):
N8N_WEBHOOK_URL=https://tu-dominio.com

# n8n arranca automГЎticamente con:
export WEBHOOK_URL=https://tu-dominio.com
export N8N_PROTOCOL=https
export N8N_PROXY_HOPS=1
```

## MГіdulo: Ollama

Modelos de IA locales. Sin internet, sin costo por token, API compatible con OpenAI. Al instalar se generan automГЎticamente `vision_bot.py` y `bot_utils.py`.

```
вң… pkg install ollama (compilado para Termux ARM64)
вң… Servidor en :11434 con API compatible OpenAI
вң… Chat texto con historial SQLite persistente por chat_id
вң… Chat con imГЎgenes (visiГіn) вҖ” redimensionado automГЎtico
вң… vision_bot.py вҖ” bot de visiГіn para n8n/Telegram
вң… bot_utils.py  вҖ” helpers SQLite reutilizables
```

**Modelos recomendados (dispositivos вүҘ 8GB RAM):**

| Modelo | TamaГұo | Uso |
|--------|--------|-----|
| `qwen2.5:0.5b` | ~397MB | Texto liviano, muy rГЎpido |
| `qwen2.5:1.5b` | ~986MB | Balance velocidad/calidad |
| `gemma3:1b` | ~815MB | Texto general |
| `moondream:1.8b` | ~1.7GB | VisiГіn вҖ” anГЎlisis de imГЎgenes |

> вҡ пёҸ No usar modelos 7B o mГЎs en dispositivos mГіviles вҖ” crash por RAM insuficiente.

**SubmenГә desde el dashboard:**
```
[1] Iniciar / detener servidor
[2] Chat rГЎpido        (sin historial)
[3] Chat completo      (SQLite В· historial persistente)
```

**ВҝPor quГ© proot para n8n?**

**ВҝPor quГ© Node.js 20 en proot y v24 en Termux?**
n8n usa `isolated-vm`, que rompe en Node.js 22+. Claude Code requiere Node.js вүҘ18.
